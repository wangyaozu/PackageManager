# PackageManager 变更日志

本文件基于仓库 `gitlog.md` 的提交历史按版本号归档，概述每个版本的主要改动与新增功能。

## 2.0.4.1 — 2025-12-23

- 看板统计优先级：
  - 添加最高优先级数、最高优先级故事点、较高优先级数、
    较高优先级故事点、其他优先级数和其他优先级故事点字段
  - 添加优先级分类逻辑，支持最高、较高和其他优先级的识别
  - 扩展状态识别逻辑，支持更多进行中状态的关键词

- 看板统计界面优化：
  - 重新组织查询条件控件布局，提升界面紧凑性
  - 调整项目、迭代、人员选择器宽度与间距
  - 修改数据表格行定义，优化垂直空间利用
  - 在结果文本中增加故事点总数统计信息
  - 查询逻辑中累加总故事点数并显示在界面

- 看板统计性能优化：
  - 使用 Task.WhenAll 并行执行，提高初始化阶段的数据加载效率
  - 新增按负责人聚合的故事点拆分接口调用，减少重复的 API 请求次数，提升页面渲染速度

- 左侧导航改进：
  - 应用启动时默认选中“产品分类”，统一初始高亮状态。

  - 包管理配置重构：新增页面 `Views/PackageConfigPage`，抽象 `IPackageEditorHost` 以复用编辑/删除逻辑，更新 `Models/PackageItem` 依赖；主窗口命令改为导航到新页面。
  - 主页导航：点击“产品分类”导航到 `PackagesHomePage`，行为一致化。
  - 稳定性与交互优化：产品日志路径存在性检查与提示增强，日志查看页加载行为更稳健。

- 缺陷修复：
  - 修复“更新”和“签名取消”功能的启用状态控制不正确的问题。
